# Environment Variables

All configuration is via `.env` file at `/opt/noble-intel/.env` (production) or project root (development).

## Required Variables

| Variable | Example | Description |
|----------|---------|-------------|
| `GEMINI_API_KEY` | `AIzaSy...` | Google Gemini API key for embeddings. Get from [AI Studio](https://aistudio.google.com/apikey). Free tier: 1,500 req/min. |
| `DATABASE_URL` | `postgresql://noble:pass@127.0.0.1:5432/noble_intel` | PostgreSQL connection string. Auto-generated by `setup-vps.sh`. |
| `QDRANT_URL` | `http://127.0.0.1:6333` | Qdrant HTTP endpoint. Always localhost in production. |
| `QDRANT_API_KEY` | `abc123...` | Qdrant authentication key. Auto-generated by `setup-vps.sh`. |
| `QDRANT_COLLECTION` | `intel_signals` | Qdrant collection name. Do not change. |
| `REDIS_PASSWORD` | `xyz789...` | Redis password. Auto-generated by `setup-vps.sh`. |
| `REDIS_URL` | `redis://:password@127.0.0.1:6379/0` | Full Redis connection URL including password. |
| `INTEL_API_KEY` | `key123...` | API key for authenticating requests to the Intel API. Auto-generated by `setup-vps.sh`. |

## Optional Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `SUPABASE_URL` | — | Supabase project URL. Required only for alert pushing. |
| `SUPABASE_SERVICE_KEY` | — | Supabase service role key (bypasses RLS). Required only for alert pushing. |
| `SENTRY_DSN` | — | Sentry error tracking DSN. Leave blank to disable. |
| `INTEL_API_PORT` | `8000` | Port for the Intel API server. |
| `PROXY_URL` | — | HTTP proxy for scraper traffic (Phase 2). |

## How Each Variable is Obtained

### Auto-generated by `setup-vps.sh`
- `DATABASE_URL` — password generated with `secrets.token_urlsafe(32)`
- `QDRANT_API_KEY` — generated with `secrets.token_urlsafe(32)`
- `REDIS_PASSWORD` / `REDIS_URL` — generated with `secrets.token_urlsafe(32)`
- `INTEL_API_KEY` — generated with `secrets.token_urlsafe(32)`

### Manual Setup Required
- `GEMINI_API_KEY` — [Google AI Studio](https://aistudio.google.com/apikey) → Create API Key
- `SUPABASE_URL` — [Supabase Dashboard](https://supabase.com/dashboard) → Settings → API → Project URL
- `SUPABASE_SERVICE_KEY` — Supabase Dashboard → Settings → API → service_role key
- `SENTRY_DSN` — [sentry.io](https://sentry.io) → New Project → Python/FastAPI → Copy DSN

## Security Notes

- `.env` file should be `chmod 600` and owned by the `noble` user
- Never commit `.env` to version control (it's in `.gitignore`)
- The `SUPABASE_SERVICE_KEY` bypasses Row Level Security — never expose to clients
- All generated secrets use Python's `secrets` module (cryptographically secure)
- Secrets are saved to `/root/.noble-intel-secrets` during setup — delete after copying to password manager
